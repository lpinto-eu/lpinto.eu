Path

Background

Since I join the Computers and Telematics Engineering course, at Aveiro University, I felt in love with programming and software building. During my time at the university, I took a lot important courses that have proved to be a solid knowledge foundation, where I studied all phases of software building, and all layer of a software systems. This was done throw different programming levels using different programming languages.

For my Master's Degree thesis, I managed to work in a industry like environment, which later proved to be a smart decision. I joined a Health IT team, that was developing a software abstraction layer as a network connecting Hospitals - RTS. I was responsible for the creation of a SOAP webservices API, security configurations and semantic services for translation and transcoding.

It proved to be a job well done, and so I was hired by iUZ Technologies to do it all over again. They developed SISO a Health Business Management System part of the National Program for the Promotion of Oral Health. SISO is a Information system based on web-portals, which connectivity could be widely enlarged by exposing it through a SOAP webservices API - this sub-project is called inSISO.

After that, as part of iUZ's inter-operability team, I had the amazing opportunity to join a international team developing the openNCP IHE HL7 webservices API for the epSOS project. This was a great experience, that allowed me to work in a daily bases with people from all over europe, and most of all it took me to the 2012 edition of the IHE Connectathon . [there're pictures (: ].

Current work

In November of 2012, I searched for a new challenge, and moved to a new company - Maisis. I then joined a team at PT Inovação as an outsourcing developer.

The challenge was to build a Information System from scratch, using the latest technologies. The company had developed a fantastic framwork, to be used as foundation of any system, defining a basic modular architecture for the server-side. The framwork is a state of the art vision of the software, built on the principles of parallelism and asynchronism. This was made using Scala as main language, and leveraged already available libraries like Akka.

From the architectural point of view, we opted for a API first architecture, which defined the creation of two sub-systems. A server-side system accessible through a REST API, and a browser application in JavaScript implementing the presentation layer. As a server-side developer have been my task the development of the business-logic layer and also the persistence layer, this in conjunction with the creation of the webservices API.

Always trying to learn more and being aware of the programming trend, I realised that front-end development and front-end technologies, are becoming a huge deal. In order to expand my technical knowledge, I started this summer learning some front-end specific languages and framworks. As a case study I started a project for Animal Shelters management; now I am thinking about make this Open Source, and host it for free offering it to the Association that could benefit from it.
